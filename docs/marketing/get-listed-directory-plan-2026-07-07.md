# Get-listed plan — directories + CPA communities

**Date:** 2026-07-07 · marketing / GTM (pre-validation)
**Landing page:** `/works-with-your-stack` (live at https://duedatehq.com/works-with-your-stack)

The complement-not-replace hub is now the natural landing page for "get-listed"
outreach: it answers the exact question a CPA asks in a directory or a community
thread — _"does this replace / work with the tools I already run?"_ This doc is
the prioritized target list + reusable copy so the listings can be executed.

## Fit with our stage

We are **pre-validation** (goal = real CPAs try it, not scale). Get-listed is a
**validation channel**: people who find you in a software directory or a "what do
you use?" thread are high-intent and self-selected. So this is on-strategy now —
but with two guardrails carried from the rest of the site:

- **Only-show-shipped / no fiction.** On directory feature-checklists, do NOT tick
  "integrates with QuickBooks / TaxDome / etc." — DueDateHQ is a _source-monitoring
  layer_, not a per-tool integration. Ticking integration boxes we don't have is
  the same red line that withdrew the Product Hunt launch.
- **No astroturfed reviews.** G2/Capterra ranking is review-driven; with few users
  we earn a handful of _real_ reviews, we do not manufacture them.

## Who does what

Most of these need a **vendor account + ownership verification** (a company-email
confirmation or DNS/website check) — those steps are **Yuqi's** (I can't create or
verify accounts). What's prepared here: the target list, the assets checklist, and
drafted copy to paste in.

## Reusable listing copy (honest — shipped capability only)

**1-liner**

> Deadline-change monitoring for US CPA practices — with a source on every date.

**Short (≈45 words)**

> DueDateHQ watches official IRS and state sources for tax-deadline and rule
> changes, then shows exactly which of your clients each change affects — with the
> source attached. It's a monitoring layer that works on top of your existing
> practice-management, tax-prep, and bookkeeping stack.

**Long (≈90 words)**

> CPA teams already run their jobs, returns, and books somewhere. What's usually
> missing is knowing the moment the IRS or a state moves a deadline — and which
> clients that change touches. DueDateHQ monitors the official federal and state
> sources for deadline and rule changes, routes each one to the affected clients,
> and attaches the source excerpt for review. It complements rather than replaces
> your practice suite (TaxDome, Canopy, Karbon…), tax software (UltraTax, Lacerte,
> Drake…), and ledger (QuickBooks, Xero). Coverage: federal + all 50 states + DC.
> It supports evidence-backed review; it is not tax advice or a filing system.

**Community answer template** (for "what do you use to track deadlines?" threads —
adapt, never paste cold; lead with a genuine answer first)

> We built DueDateHQ for exactly this — it doesn't replace [their tool], it sits on
> top and watches the IRS/state sources for deadline & rule changes, then flags
> which clients are affected with the source attached. There's a page on how it
> layers onto common stacks: duedatehq.com/works-with-your-stack. Happy to hear if
> it's useful or missing something.
