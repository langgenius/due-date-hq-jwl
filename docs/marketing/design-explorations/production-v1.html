Questions

The short answers.

How is this different from File In Time?
File In Time records and rolls forward the deadlines you enter. DueDateHQ watches them for changes — and the moment the IRS or a state moves one, it shows you exactly which clients it affects, with the official source attached. It's deadline-change monitoring, not a deadline table.
Do I have to leave TaxDome / Karbon / Canopy?
No. DueDateHQ is deliberately narrow — the weekly deadline-triage layer, not a practice-management platform. It's email-first with no portal to configure, so it sits alongside whatever you already run.
How do I know the dates are right?
Because nothing is the source of truth except the official source. Every date keeps its IRS or state source link, an exact quote, and a verified date — and anything that can't be grounded in the source is held back, not shown. It turns the official notice into something you can act on; it never invents a date.
