# PR 117 CI format and lint repair

GitHub Actions run `29486084845` failed in `CI / ci` while running `vp run ci` on `main` at
`61036f4e4`.

## Root cause

- `vp check` found format drift in 58 files added or regenerated by the outreach, disaster-alert,
  digest, and CPA Field Guide work.
- After formatting, lint found a `no-map-spread` violation in the disaster-notices JSON feed and an
  unused `scope` variable in the alert email builder.
- The app entry bootstrap test dynamically imports the complete entry module but inherited Vitest's
  five-second default timeout. Under full-workspace parallel transform load, the import repeatedly
  completed just after that limit even though its assertions passed in focused and app-only runs.

## Repair

- Applied the repository formatter to the flagged source, docs, JSON, email preview, digest, and
  checked-in CPA Field Guide deploy artifacts.
- Kept optional feed fields as normal object properties. `JSON.stringify` continues to omit their
  `undefined` values without allocating conditional spread objects inside `map`.
- Removed the unused alert-email variable.
- Set a 15-second timeout only on the integration-style app entry test; its behavior assertions and
  the global Vitest timeout are unchanged.

## Validation

- `pnpm check`
- `pnpm --filter @duedatehq/app test src/main.test.tsx`
- `pnpm --filter @duedatehq/app test`
- `pnpm run ci` (two consecutive full-workspace passes)
- `git diff --check`

Two full-workspace runs reproduced the app entry import timeout, while the focused test and complete
app suite passed. The explicit per-test timeout covers workspace contention without relaxing the
global timeout or changing application behavior.

No `DESIGN.md` or `docs/dev-file/` update was needed because this repair restores existing CI format
and lint contracts without changing product behavior or architecture.
