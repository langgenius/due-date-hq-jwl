# Pass-2 P1 defects, round 2 (error-as-empty + dark contrast + kbd)

_2026-06-18_

More net-new P1s from the pass-2 audit (the 8 rate-limited lenses re-ran and
surfaced these). All real defects.

## Error-as-empty (the `throwOnError:false` root cause, in routes pass 1 didn't touch)

- **`needs-attention-section.tsx`** — the dashboard's PRIMARY alert section gated only
  on `isLoading`; on a failed alert load it rendered the calm "You're all caught up"
  state — a false all-clear on the most prominent surface. Added an `isError` branch
  (destructive Alert + Retry).
- **`reminders-page.tsx`** — templates + recent-sends queries fell to empty on error.
  Added per-panel error branches (shared `RemindersErrorState` + Retry).
- **`obligations.tsx`** — the signature-reminder preview dialog showed a misleading
  "No email address on file…" and the bulk-extension preview an all-zeros dead-end on
  error. Added `isError` branches (inline message + Retry) to both.

## Dark-mode contrast (pinned text on a mode-shifting accent surface)

- **`AlertDetailDrawer.tsx`** Confirm button + **`AlertsListPage.tsx`** selection check:
  `text-white` on `bg-state-accent-solid` (which shifts to the lighter `primary-400` in
  dark → white fails AA) → `text-text-primary-on-surface` (the adaptive pair the
  accent-solid shift was designed for; white in light, near-black in dark).

## Design-system

- **`rule-detail-drawer.tsx`** — the last live raw `<kbd>` keycap → the `Kbd` primitive.
