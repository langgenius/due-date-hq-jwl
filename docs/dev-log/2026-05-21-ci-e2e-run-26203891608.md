## Context

GitHub Actions CI run `26203891608` failed in the secret scan. E2E run `26203891625`
failed after the app navigation, dashboard sections, rules library route, obligations controls, Pulse
review card, client detail content, and authenticated 404 behavior moved ahead of the browser tests.

Local E2E repro also exposed Wrangler dev instability: the worker watched and reloaded the default
local `.wrangler/state` directory while Playwright was creating and migrating isolated test data,
which caused transient `503` responses and process spawn failures.

## Changes

- Replaced Gitleaks false-positive sample identifiers in the CPA/SaaS competitor export research
  report with explicit placeholder strings.
- Moved Playwright's local Wrangler state to a per-run temp `--persist-to` directory for both D1
  migration apply and `wrangler dev`.
- Added auth-session retry handling for short-lived local worker socket failures during E2E setup.
- Realigned E2E page objects and specs with the current Today shell, weekly Actions dashboard,
  Clients page, Obligations drawer, Rule library, Pulse review flow, Workload counts, Opportunities
  copy, and authenticated not-found route.
- Adjusted the Pulse E2E seed so Pulse matching uses the legal due date as `currentDueDate`; the
  obligation queue still renders the internal deadline after the alert is applied.

## Validation

- `gitleaks detect --source docs/report --no-git --redact=100` passed with no leaks found.
- `pnpm exec playwright test --project=chromium --workers=1 --retries=0` passed 60 tests in 2.0m.
- `pnpm check` passed with 7 existing warnings unrelated to this CI/E2E repair.

## Docs and Design Alignment

No `DESIGN.md` or stable architecture spec update was needed: this change only refreshes test
contracts, local E2E harness isolation, seed data consistency, and report placeholder literals.
